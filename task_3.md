# Task 3: SDFT on Synthetic Algebra Adapter

Train an adapter model using the SDFT method (reverse KL distillation) on the synthetic algebra dataset. The teacher sees the explicit symbol mapping; the student only gets a vague description and must learn the mapping through distribution matching.

## Models (from api-adapter-ak)

| Role | Model | Notes |
|------|-------|-------|
| Student | `Qwen/Qwen3-8B` + LoRA (rank=32, alpha=64) | Same as api-adapter-ak adapter |
| Teacher | `Qwen/Qwen3-8B` + LoRA (rank=32, alpha=64) | EMA-tracked copy of student |
| Inference | `Qwen/Qwen3-8B` + LoRA | vLLM server for rollouts |

## Dataset

Generated by `synthetic_algebra_dataset.py` at repo root:
- `data/train.jsonl` (1600 samples: 800 custom + 800 standard)
- `data/test.jsonl` (400 samples: 200 custom + 200 standard)

Schema per row:
```json
{"expression": "13 γ 46 θ 45", "answer": 643, "type": "custom"}
{"expression": "31 + 73", "answer": 104, "type": "standard"}
```

## Prompt format (adapter-style)

Both student and teacher see the adapter task: check the API answer and output CORRECT or the computed answer.

### Student prompt (vague symbols)
```
System: You are an adapter that checks an API model's arithmetic answer.
        If correct, respond with \boxed{CORRECT}.
        If wrong or missing, compute the correct answer: \boxed{answer}.

User: The symbols θ, α, γ, β each represent one of the four basic arithmetic
      operations (+, -, ×, ÷). Each symbol maps to exactly one operation.
      Standard operator precedence (BODMAS) applies.

Examples:
Expression: 3 θ 4 | API answer: 7 → \boxed{CORRECT}
Expression: 10 α 3 | API answer: 5 → \boxed{7}
Expression: 2 γ 6 | API answer: none → \boxed{12}

Expression: {expr} | API answer: {api_answer} →
/no_think
```

### Teacher prompt (explicit symbol definitions)
Same as student, but replaces the vague symbol description with:
```
Custom arithmetic symbols:
  θ means addition (+)
  α means subtraction (-)
  γ means multiplication (×)
  β means division (÷)
Precedence: β and γ bind tighter than θ and α.
```

## How SDFT applies here

1. **Teacher has privileged info**: the explicit symbol → operation mapping (student only gets the vague "each symbol maps to one operation" hint)
2. **Student generates a completion** (via vLLM rollout): the adapter output (CORRECT or computed answer)
3. **Teacher computes log-probs** on `[conditional_prompt + completion]`: teacher sees the same completion but with explicit symbol knowledge — its distribution reflects the correct answer
4. **Reverse KL loss**: `KL(p_student || p_teacher)` — the student is pulled toward the teacher's distribution on the completion tokens
5. **Weight sync**: student weights → teacher (EMA) + vLLM after each optimizer step

No reward function needed — the teacher's log-prob distribution itself encodes correctness.

### Key config differences from MaaS SDFT
| | MaaS SDFT | Adapter SDFT |
|---|---|---|
| Prompt format | Chat (Q&A) | Adapter (check + correct) |
| Privileged info | Documentation excerpt | Symbol → op mapping |
| Completion | Full answer text | `\boxed{CORRECT}` or `\boxed{N}` |
| Max tokens | 2048 | 64 (answers are short) |
| Eval metric | Loss | Accuracy (custom + standard) |
| LoRA | No | Yes (rank=32, alpha=64) |

## Evaluation

Run adapter on test set:
- Parse `\boxed{CORRECT}` → API answer; `\boxed{N}` → computed answer
- Compare to ground truth
- Metrics: custom accuracy, standard accuracy, overall accuracy

Baselines to beat:
- Claude Haiku (no adapter): 0% custom, 75.5% standard
- GRPO adapter @ 1000 steps: 84% custom, 97.5% standard
- Untrained Qwen3-8B: ~10-15% custom (random guessing)

## Experiment variants (ablations)

1. **Condition D (vague + CORRECT)**: student gets vague symbols + CORRECT token — primary experiment
2. **Condition C (explicit + CORRECT)**: student gets explicit symbol defs — sanity check, should hit 100% immediately
3. **No CORRECT token**: student must always compute, no pass-through — tests if model can learn to compute always
4. **No few-shot examples**: remove the example lines from the prompt — tests if model can learn purely from KL signal
