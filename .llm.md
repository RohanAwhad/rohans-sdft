What we are doing here: Self-Distillation Fine Tuning.

- The way to do that is we take input qquestion, we use the inference engine to rollout the generation
- we take the generation and in the training process we get student logprobs
- On the teacher side, we get logprobs from the logprobs server we created in task 2.
  - The teachers context is : question + privileged information + student generated answer 
- Then compute reverse KL on generated answer's token ids and backprop on student.

- After the epoch, push weights to inference engine and logprob generator server.
